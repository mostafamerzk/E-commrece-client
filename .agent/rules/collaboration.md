# Project Collaboration & Memory Rule

## Core Principle
- The file `docs/memory.md` is the "Source of Truth" for project state.
- Always check `docs/memory.md` before suggesting changes to ensure context is current.
- If a major architectural decision is made during a chat, remind the user to update `docs/memory.md`.

## Session Protocol
- At the start of a session, summarize the current task from `docs/memory.md`.
- Ensure all code follows the patterns defined in the `Standard Development Patterns` section.
